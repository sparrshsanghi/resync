# Resync AI

Resync AI is a FastAPI backend that recommends YouTube learning videos, generates a step-by-step learning roadmap, and tracks session-based weakness/unsolved-question items.

## Features

- Search YouTube and rank relevant videos for a learning goal
- Generate a structured learning roadmap with step metadata
- Track roadmap progress and completed steps
- Store unsolved questions, weakness tags, and review status per session
- Expose clean API endpoints for roadmap and weakness management

## Prerequisites

- Python 3.12
- A virtual environment (recommended)
- `GROQ_API_KEY` if you want LLM-powered roadmap generation and search query generation

## Setup

```bash
cd /home/sparrsh/Desktop/resync
python3 -m venv .venv
source .venv/bin/activate

```

## Configuration

Create a `.env` file or set environment variables directly. Supported variables:

- `GROQ_API_KEY` — required for LLM calls
- `SKIP_TRANSCRIPTS` — set to `true` to skip YouTube transcript fetching

## Run Locally

```bash
source .venv/bin/activate
.venv/bin/python -m uvicorn main:app --host 0.0.0.0 --port 8000
```

Then open:

- `http://127.0.0.1:8000/health`
- `http://127.0.0.1:8000/docs`

## API Endpoints

### Recommendation and roadmap

- `POST /recommend`
  - Request body: `goal`, optional `user_id`, optional `max_videos`
  - Returns: recommended videos, generated roadmap, and `session_id`

- `POST /next-step`
  - Request body: `user_id`, `completed_steps`
  - Returns: next roadmap step and progress

- `GET /roadmap/{session_id}`
  - Returns full roadmap with completion status and progress

- `POST /roadmap/{session_id}/complete`
  - Request body: `completed_steps`
  - Marks roadmap steps complete and returns updated progress

- `GET /roadmap/{session_id}/progress`
  - Returns roadmap progress summary for the session

- `POST /roadmap/regenerate`
  - Request body: `session_id`, optional `goal`
  - Regenerates the roadmap and resets completed step tracking

### Weakness analysis

- `POST /weakness/{session_id}/unsolved`
  - Request body: `question`, optional `notes`, optional `tags`
  - Stores a new unsolved question for the session

- `GET /weakness/{session_id}`
  - Optional query parameter: `status` (`unsolved`, `reviewed`, `resolved`)
  - Returns all weakness items for the session with counts by status

- `POST /weakness/{session_id}/resolve`
  - Request body: `question_id`, `status`
  - Updates the status of an existing weakness item

## Notes

- Session state is stored in memory only. If the app restarts, sessions and weakness items are lost.
- `session_id` is generated by `/recommend` and should be reused for roadmap and weakness tracking.

## Deployment

### Local

This repository uses FastAPI and Uvicorn. For local deployment, run:

```bash
source .venv/bin/activate
.venv/bin/python -m uvicorn main:app --host 0.0.0.0 --port 8000
```

### Deploy to Render

1. Push this repository to GitHub
2. Go to [render.com](https://render.com) and sign up
3. Create a new **Web Service**
   - Connect your GitHub repository
   - Select this repository
4. In the Render dashboard:
   - **Name**: `resync-ai` (or your choice)
   - **Environment**: Python
   - **Region**: choose closest to you
   - **Build Command**: `pip install -r requirements.txt`
   - **Start Command**: `uvicorn main:app --host 0.0.0.0 --port $PORT`
5. Under **Environment Variables**, add:
   - `GROQ_API_KEY`: paste your Groq API key
   - `SKIP_TRANSCRIPTS`: set to `true` to skip YouTube transcript fetching
6. Click **Create Web Service**

Render will automatically build and deploy. Your app will be available at `https://<service-name>.onrender.com/`.

The `render.yaml` and `Procfile` in this repo support automatic deployment configuration.
